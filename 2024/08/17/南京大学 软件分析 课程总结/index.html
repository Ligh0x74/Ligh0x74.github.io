Course Introduction

概念

程序设计语言（Programming Languages，PL）可以分为三个部分，Theory、Environment 和 Application。静态程序分析（Static Program Analysis）属于 PL 的 Application 部分。

Rice’s Theorem: Any non-trivial property of the behavior of programs in a r.e. language is undecidable.

不存在完美的静态分析，满足 Sound（Overapproximate）且 Complete（Underapproximate）。

大多数静态分析可以使用两个词概括：Abstraction + Over-approximation（Transfer functions，Control flows）。

问题

Q：为什么需要静态分析？什么是静态分析？

Q：Sound 和 Complete 是什么意思？误报（false positives）和漏报（false negatives）的权衡？

Q：如何使用一句话概括静态分析？什么是有用的静态分析？

A：ensure (or get close to) soundness, while making good trade-offs between analysis precision and analysis speed.

Q：如何理解 Abstraction 和 Over-approximation？

A：首先对数据做抽象，然后对数据流做近似。近似涉及到 Transfer function 如何设计和 Control flows 如何合并。如果是 Over-approximation，就会使结果接近 Sound。

Intermediate Representation

概念

编译器（Compiler）对源代码编译的过程可以分为三个阶段：

（1）前端：使用正则表达式进行 Lexical Analysis 得到 Tokens，之后使用 Context-free Grammar 进行 Syntax Analysis 得到 AST，然后使用 Attribute Grammar 进行 Semantic Analysis 得到 Decorated AST，然后 Translator 将 Decorated AST 转换为 Intermediate Representation（IR）。此处的 IR 通常指 Three-Address Code（3AC）。

（2）中端：静态分析器会根据 3AC 构建 Control Flow Graph（CFG），CFG 以 Basic Block（BB）作为节点，以控制的转移作为边。然后静态分析器基于 CFG 进行静态分析，最后将优化之后的 IR 交给 Code Generator。

（3）后端：Code Generator 将 IR 转换为 Machine Code。

问题

Q：为什么 IR 使用 3AC，而不直接使用 AST？

A：AST 和语言相关，3AC 和语言无关。使用 3AC 作为 IR，使得中端的优化更加通用，能够在不同前端和后端共享。而且 3AC 包含控制流信息，更有利于静态分析，等等。

Q：如何基于 IR 构建 BB？如何基于 BB 构建 CFG？

Data Flow Analysis - Applications

概念

may analysis（over-approximation），must analysis（under-approximation）。

Each execution of an IR statement transforms an input state to a new output state.

The input (output) state is associated with the program point before (after) the statement.

In each data-flow analysis application, we associate with every program point a data-flow value that represents an abstraction of the set of all possible program states that can be observed for that point.

Data-flow analysis is to find a solution to a set of safe-approximation-directed constraints on the IN[s]’s and OUT[s]’s, for all statements s.

  • constraints based on semantics of statements (transfer functions)

  • constraints based on the flows of control

数据流分析的应用：

  • Reaching Definitions Analysis (may analysis)

    A definition d at program point p reaches a point q if there is a path from p to q such that d is not “killed” along that path.

  • Live Variables Analysis (may analysis)

    Live variables analysis tells whether the value of variable v at program point p could be used along some path in CFG starting at p. If so, v is live at p; otherwise, v is dead at p.

  • Available Expressions Analysis (must analysis)

    An expression x op y is available at program point p if (1) all paths from the entry to p must pass through the evaluation of x op y, and (2) after the last evaluation of x op y, there is no redefinition of x or y.

问题

Q：什么是 safe-approximation？

A：取决于具体使用场景，在大多数场景 Sound 就是 safe-approximation，而小部分场景 Complete 才是。

Q：为什么 Reaching Definitions Analysis 的算法最终会停止？

A：所有的 OUT 对应的 Bit Vector，它包含的 1 的数量都是单调递增的，最后必定会到达一个上界。

Q：为什么 Live Variables Analysis 使用 Backward Analysis，而不是 Forward Analysis？

A：从算法设计的角度看，计算变量是否存活，使用 Backward Analysis 的时间和空间更优，它只需要迭代。而使用 Forward Analysis 需要使用递归。
